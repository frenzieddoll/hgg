# Layers & Marks тАФ Reference

> ЁЯМР **English** | [цЧецЬмшкЮ](02-layers.ja.md)

> [ЁЯУЪ Index](README.md) | [01 quickstart](01-quickstart.md) | **02 layers** | [03 encoding & scale](03-encoding-scale.md) | [04 decoration](04-decoration.md) | [05 backends](05-backends.md) | [06 dataframe](06-dataframe.md) | [07 analyze](07-analyze.md) | [08 3d](08-3d.md) | [09 appendix](09-appendix.md)

The composition unit is **layer**, the drawing type is **mark** (type name `MarkKind`), and individual functions are `scatter` / `line` / `bar` / тАж.
Marks are composed as `layer (mark <> modifiersтАж)` stacked onto a plot. Arguments of type `ColRef` are **`inline [..]`** (numeric) / 
**`inlineCat [..]`** (categorical), or column name literals like **`"weight"`** when using DataFrames ([06 dataframe](06-dataframe.md)).

This page focuses on **a catalog of drawable plots (marks)**. Structure: **[1. Mark index by role](#index)** (one row per mark / 
one-hop to details) тЖТ **[2. Mark standard entries](#entries)** (uniform format per mark).

> All marks share common appearance and channel modifiers (`colorBy` / `size` / `shape` / `position` / fixed color `Color` etc.)
> and scale / axis control, documented in [03 encoding & scale](03-encoding-scale.md). The "encoding" section of each standard entry refers
> to modifiers on that page.

> For ggplot2 users: this library's **mark** corresponds to ggplot's `geom_*` (e.g. `scatter` =
> `geom_point`). Since "geom" is ggplot dialect absent from Wilkinson's Grammar of Graphics, this reference uses
> native **mark / layer** terminology, and "geom" is referenced only for cross-reference with ggplot.

> Figures: Minimal examples with figures are being progressively supplemented. Entries without figures are complete in specification
> via signature, encoding, and code examples. Remaining figures are generated by `cabal run doc-figures` for later addition.

> **See examples**: Many mark examples with real data are in the [README gallery](../../README.md)
> (click to navigate to each entry) and [R for Data Science Chapter 1](../tutorials/01-visualize/README.md)
> (24 penguins plots + reproducible code).

### `scatter` / `line` / `step` тАФ Points / Line / Step line

**Signature** `scatter, line, step :: ColRef -> ColRef -> Layer`

**What it draws** (x, y) as points (`scatter`) / line connecting in order (`line`) / stepped line (`step`). Basic marks for continuous ├Ч continuous.
All three share arguments and encoding.

**Encoding**
- Required: x, y (two `ColRef`)
- Optional: [`colorBy`](03-encoding-scale.md#encoding) `colorContinuousBy` `sizeBy` `shapeBy` `alpha` `linetype` (`line`/`step`) `jitterX` / `jitterY` `connect`

**Minimal example**

```haskell
purePlot <> layer (scatter (inline [1,2,3,4,5]) (inline [2,4,3,5,7]))
-- To color by group, add <> colorBy (inlineCat [...]) ([encoding channel](03-encoding-scale.md#encoding))
```

![scatter](images/scatter.svg)

**Related / ggplot** [`scatterPoints` / `linePoints`](#e-points) (direct `[Point2]`) | Overplot handling via [`jitterX`](03-encoding-scale.md#encoding) |
ggplot: `geom_point` / `geom_line` / `geom_step`

<a id="e-bar"></a>

### `bar` тАФ Bar chart

**Signature** `bar :: ColRef -> ColRef -> Layer`

**What it draws** Bar chart of (categorical x, numeric y). Grouping and stacking controlled via `position`.

**Encoding**
- Required: x (categorical) / y (height)
- Optional: [`colorBy`](03-encoding-scale.md#encoding) `color` `alpha` `position`

**Options** (within mark via `<>`)

| Option | Type | Meaning | Default |
|---|---|---|---|
| `position` | `Position -> Layer` | `PosIdentity` / `PosDodge` (side-by-side) / `PosStack` (stacked) / `PosFill` (100%) | `PosStack` |

**Minimal example**

```haskell
purePlot <> layer (bar (inlineCat ["A","B","C"]) (inline [3,7,5])) <> xLabel "group" <> yLabel "value"
-- Group ├Ч series side-by-side (dodge)
let bx = inlineCat ["A","A","B","B","C","C"]; bg = inlineCat ["g1","g2","g1","g2","g1","g2"]
    bv = inline [3,2,5,4,4,6]
in purePlot <> layer (bar bx bv <> colorBy bg <> position PosDodge) <> legend
```

| `bar` | `bar <> colorBy <> position PosDodge` |
|---|---|
| ![bar](images/s3a-geom.svg) | ![bar dodge + color](images/s3b-aes.svg) |

**Related / ggplot** [`stem`](#e-stem) (lollipop) | [`histogram`](#e-histogram) (binned frequency) | [`waterfall`](#e-waterfall) (cumulative contribution) |
ggplot: `geom_col` / `geom_bar(stat="identity")`

### `text` / `label` тАФ Text / Label

**Signature** `text, label :: ColRef -> ColRef -> ColRef -> Layer`

**What it draws** Text strings at (x, y, label column) positions. `text` is plain text, `label` has a background frame.

<a id="dist-compose"></a>

### `(<+>)` / `distCols` тАФ Compose / Juxtapose Distribution Marks

**Signature** `(<+>) :: Layer -> Layer -> Layer` я╝П `distCols :: [Layer] -> VisualSpec`

**What it draws** `<+>` bundles distribution marks into a single `Layer`. Position (horizontal slot) is determined by the **value column of each mark**:
**same value column**тЖТoverlay (`raincloud` is this preset = `violin <+> box <+> strip`), **different value column**тЖТside-by-side
(= `distCols` / column names become x-axis labels). `distCols [a,b,c]` is syntactic sugar for `a <+> b <+> c` wrapped in `layer`, allowing mixed marks while sharing y-axis.
Lanes in a composed layer are queried via `compositeLanes :: Layer -> [ColRef]`.

**Encoding** Applying [`colorBy`](03-encoding-scale.md#encoding) to each lane (component Layer) splits and colors that lane by group
(reusing existing dodge mechanics). Nudging via `nudge`.

**Minimal example**

```haskell
distCols [ boxplot "a", violin "c", boxplot "d" ] <> yLabel "value"   -- Different columns, different marks, juxtaposed
distCols [ boxplot "a" <> colorBy "g", boxplot "c" ] <> legend         -- Lane with colorBy for dodge
```

| `distCols` juxtapose | `distCols ├Ч colorBy` |
|---|---|
| ![distCols juxtapose](images/distcols.svg) | ![distCols ├Ч colorBy](images/distcols-colorby.svg) |

> **Limitation**: Applying `<> groupBy "g"` (non-color grouping) to a lane is currently ignored (lane position determined by name).
> Use `<> colorBy "g"` for group splitting. Using different `colorBy` columns across lanes creates ambiguous legend.

**Related / ggplot** Overlay preset is [`raincloud`](#e-raincloud) | Standalone distributions: [`boxplot`](#e-boxplot) / [`violin`](#e-violin) |
ggplot: Multiple `geom_*` + `position_nudge`

<a id="e-range"></a>

### `lineRange` / `pointRange` / `crossbar` тАФ Interval

**Signature** `lineRange, pointRange, crossbar :: ColRef -> ColRef -> ColRef -> Layer`

**What it draws** Symmetric interval `y ┬▒ err` from (x, y, err). `lineRange` is vertical line, `pointRange` has center point,
`crossbar` is box-shaped interval.

**Encoding** Required: x, y (center), err (symmetric half-width) я╝П Optional: [`colorBy`](03-encoding-scale.md#encoding) `color`

**Minimal example**

```haskell
-- (x, center y, symmetric half-width err)
let c = inlineCat ["A","B","C"]; y = inline [2,3,2.5]; e = inline [0.4,0.6,0.3]
in subplots [ layer (lineRange  c y e) <> title "lineRange"
            , layer (pointRange c y e) <> title "pointRange"
            , layer (crossbar   c y e) <> title "crossbar" ]
   <> subplotCols 3
```

### `dag` / `dagFromLists` / `dagFromListsWithPlates` тАФ Probabilistic Model Structure Diagram

**Signature** `dag :: [DAGNode] -> [DAGEdge] -> Layer` (= `dagFromLists тАж LayoutManual`)

**What it draws** DAG from nodes and edges listed directly. Normally auto-generated from HBM via [`dagOf`](07-analyze.md#hbm-plotting),
but can be assembled manually.

**Encoding** Required: `[DAGNode]` / `[DAGEdge]` я╝П Optional: `size` (node diameter)

**Options / Related Constructors**

| Function | Type | Purpose |
|---|---|---|
| `dagFromLists` | `тАж -> DAGLayoutAlgorithm -> Layer` | Layout specification (`LayoutManual` / `LayoutHierarchical`) |
| `dagFromListsWithPlates` | `тАж -> [DAGPlate] -> Layer` | With plate frames (repeated structure) |
| `dagNode` | `Text -> Text -> DAGNodeKind -> Double -> Double -> DAGNode` | Node (id, label, kind, x, y) |
| `dagNodeDist` | `тАж -> Text -> Double -> Double -> DAGNode` | Node with distribution name |
| `dagEdge` | `Text -> Text -> DAGEdge` | Edge (from-id тЖТ to-id) |

`DAGNodeKind` = `NodeLatent` (white oval) / `NodeObserved` (gray oval) / `NodeDeterministic` (white rectangle) /
`NodeData` / `NodeOther` (PyMC `model_to_graphviz` convention). Coordinates in `LayoutManual` mode normalized to `[0,1]` space.
`DAGPlate` is a record: `DAGPlate { dpLabel = "obs (N)", dpNodeIds = ["mu","y"] }`.

**Minimal example**

```haskell
import Graphics.Hgg.Spec   -- dag, dagNode, dagNodeDist, dagEdge, DAGNodeKind (..), ...

let nodes = [ dagNodeDist "a"  "a"  NodeLatent        "Normal(0,10)"  0.15 0.0
            , dagNodeDist "b"  "b"  NodeLatent        "Normal(0,10)"  0.45 0.0
            , dagNodeDist "s"  "s"  NodeLatent        "HalfNormal(1)" 0.85 0.0
            , dagNode     "mu" "mu" NodeDeterministic               0.30 0.5
            , dagNodeDist "y"  "y"  NodeObserved      "Normal(mu,s)"  0.45 1.0 ]
    edges = [ dagEdge "a" "mu", dagEdge "b" "mu", dagEdge "mu" "y", dagEdge "s" "y" ]
purePlot <> layer (dag nodes edges <> size 22) <> title "Manual DAG"
```

|---|---|

**Related / ggplot** Auto-generation from HBM: [`dagOf`](07-analyze.md#hbm-plotting) | Related types: `DAGNode` / `DAGEdge` /
`DAGPlate` / `DAGSpec` / `DAGNodeKind` / `DAGLayoutAlgorithm` | `edge :: Text -> Layer` ([encoding](03-encoding-scale.md#encoding)) is different
