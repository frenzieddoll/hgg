> ⚠️ **3D uses a separate type system from 2D**. Composition unit is `Layer3D`, plots are `VisualSpec3D`, marks are `Mark3DKind`
> (`scatter3D` / `surface3D` / …), saving via `saveSVG3D` etc. Layer composition (`purePlot3D <> layer3D (…)`) works the same,
> but 2D `Layer` / `VisualSpec` ([02 layers](02-layers.md)) and 3D cannot be mixed.

`hgg-3d` builds 3D using the same spec API as 2D (`purePlot3D <> layer3D (...)`) with CPU projection backend (equivalent to mplot3d).
Beyond response surface (RSM) plots with continuous color, floor contours, overlaid observations, and grouped panels, it delivers **all general-purpose features** (group color scatter, bubbles, 3D bars, text annotations, pane walls, box aspect, log axes) in one path.

## Core marks / builders—types {#core-3d}

Arguments `ColRef` are either `inline [..]` or column names (when using df). `grid :: [[Double]]` has rows = y gridlines.

### Marks (`Layer3D`)

| Function | Type | Purpose |
|---|---|---|
| `scatter3D` / `line3D` / `stem3D` / `bar3D` | `ColRef -> ColRef -> ColRef -> Layer3D` | Scatter / line / stem / bar (x, y, z) |
| `scatter3DPoints` / `line3DPoints` / `bar3DPoints` / `stem3DPoints` / `trisurf` | `[Point3] -> Layer3D` | Each mark from point list |
| `surface3D` / `surface3DGrid` | `ColRef -> ColRef -> ColRef -> Layer3D` / `[[Double]] -> Layer3D` | Long (x, y, z columns) / grid matrix response surface |
| `wireframe3D` | `[Point3] -> [(Int, Int)] -> Layer3D` | Wireframe (vertices, edges) |
| `quiver3D` | `[(Point3, Vec3)] -> Layer3D` | 3D vector field |
| `text3D` / `text3DPoints` / `annotate3D` | `ColRef -> ColRef -> ColRef -> ColRef -> Layer3D` / `[(Point3, Text)] -> Layer3D` / `Point3 -> Text -> Layer3D` | (x, y, z, label column) / inline point labels / single point annotation |

### Mark options (`Layer3D`, within mark, use `<>`)

| Function | Type | Purpose |
|---|---|---|
| `color3D` | `Color -> Layer3D` | Solid color (`color3D (fromHex "#..")`) |
| `colorRGBA3D` | `Text -> Layer3D` | 8-digit RGBA hex shortcut (= `color3D (fromHex …) <> alpha3D …`, sibling to 2D `colorRGBA`) |
| `colorRGBA3DMaybe` | `Text -> Maybe Layer3D` | Total `colorRGBA3D`. Invalid hex → `Nothing` |
| `colorBy3D` / `colorContinuousBy3D` / `sizeBy3D` | `ColRef -> Layer3D` | Group color / value continuous color / bubble (column) |
| `size3D` / `alpha3D` / `width3D` / `vecScale3D` | `Double -> Layer3D` | Point size / transparency / line width / arrow scale |
| `sizeRange3D` / `xRange3D` / `yRange3D` | `(Double, Double) -> Layer3D` | Bubble radius / x range / y range |
| `colormap3D` / `edgeOn` | `Layer3D` | Surface/trisurf z continuous color / face edges (no args) |
| `colormapWith3D` | `[Text] -> Layer3D` | Explicit colormap as color list |
| `contourZ` / `contourX` / `contourY` | `Int -> Layer3D` | Projected contour count (floor=z level set / left/right wall=x cross-section / front/back wall=y cross-section, matplotlib `zdir` equivalent, composable with surface, projected walls auto-fixed far from camera) |
| `surfaceWire` | `Layer3D` | Draw surface as **gridlines without fill** (matplotlib `plot_wireframe` equivalent, no args, line color via `color3D`) |
| `shaded3D` | `Bool -> Layer3D` | Surface/trisurf faces with Lambert shading (wireframe edges only, no effect) |
| `edgeColor3D` | `Color -> Layer3D` | Edge color |
| `errorBar3D` | `ColRef -> Layer3D` | Error bar column |
| `barStyle3D` / `barWidth3D` / `stemBaseZ` | `BarStyle3D -> Layer3D` / `Double -> Layer3D` / `Double -> Layer3D` | Bar style / width / stem base z |

### 7-1. Surface + colormap + projected contour

Add **`colormap3D`** (z-value viridis continuous color) and **`contourZ n`** (floor projection isoline, plotly contours_z / matplotlib `zdir='z'` equivalent) to `surface3DGrid grid` with `<>`. `saveSVG3D` auto-adds colorbar.

```haskell
import Hgg.Plot.ThreeD.Spec
import Hgg.Plot.ThreeD.Easy  (saveSVG3D)
import Hgg.Plot.ThreeD.Types (cameraIso)

saveSVG3D "surface.svg"
  ( layer3D ( surface3DGrid grid              -- grid :: [[Double]] (rows = y)
           <> xRange3D (-3, 3) <> yRange3D (-3, 3)
           <> colormap3D                  -- z continuous color + colorbar
           <> contourZ 8 )                -- Project 8 z isosurfaces to floor
 <> camera (cameraIso 5)                  -- Viewpoint preset (iso/top/front/side)
 <> title3D "3D surface + floor contour" )
```

![3D surface + floor contour](images/3d/surface3d-sinc-floor-contour.svg)

**Wall-projected contour (`contourX` / `contourY`)** — matplotlib `contour(..., zdir='x'/'y')` / plotly `contours.x.project` equivalent.
`contourX n` divides the x-axis into n positions, cuts the surface at each, and draws each **cross-section profile** `z = f(x_k, y)` on left/right walls (`contourY` draws y sections on front/back walls).
**Compose with the same surface using `<>`** — projected walls **auto-fix to far side from camera**
(floor `contourZ` = level set overlays, surface shape readable from both top and side).

```haskell
saveSVG3D "wall.svg"
  ( layer3D ( surface3DGrid grid <> xRange3D (-3,3) <> yRange3D (-3,3) <> colormap3D
           <> contourX 8 <> contourY 8 <> contourZ 8 )  -- Compose all 3 directions in 1 layer
 <> axes3D wallAxes                        -- Wider axes than data looks cleaner
 <> camera (cameraIso 5) )
```

![3D surface + wall-projected contour](images/3d/surface3d-wall-contour.svg)

**Unfilled gridline mesh (wireframe surface)** — Add `surfaceWire` with `<>` to draw grid rows/columns as line mesh without fill (matplotlib `plot_wireframe` equivalent).
Uses the same grid/range as `surface3D`, so filled vs wireframe toggles with one flag. Line color via `color3D` (colormap/shaded for faces only, no effect):

```haskell
saveSVG3D "wire.svg"
  ( layer3D ( surface3DGrid grid <> surfaceWire <> color3D (fromHex "#2563eb")
           <> xRange3D (-3, 3) <> yRange3D (-3, 3) )
 <> camera (cameraIso 5) )
```

![surfaceWire (gridline mesh surface like plot_wireframe)](images/3d/surface3d-wire.svg)

> ⚠️ Different from `wireframe3D` (cube at [7-8](#more-3d)). `surfaceWire` **auto-generates row/column line mesh from grid**
> (surface wireframe); `wireframe3D` **takes arbitrary vertices + edges** by hand (general-purpose lines).

### 7-4. Analyze integration — Surface from fitted models (cross-repo)

`Hanalyze.Plot`'s `surfaceOf` / `surfaceGrid` let you turn fitted quadratic models (RSM) into surfaces by grid-evaluating two factors (other factors fixed via `HoldAgg`).
Polynomial terms like `I(x1^2)` recalculate at grid points, so true curvature appears.

```haskell
import Hanalyze.Plot (multiLMModel, surfaceGrid, defaultSurfaceOpts)

let m = either error id
          (multiLMModel "y ~ x1 + x2 + I(x1^2) + I(x2^2) + I(x1*x2)" df)
    (gxs, gys, grid) = surfaceGrid m "x1" "x2" defaultSurfaceOpts
saveSVG3D "rsm.svg"
  ( layer3D (surface3DGrid grid <> xRange3D (head gxs, last gxs)
                            <> yRange3D (head gys, last gys)
                            <> colormap3D <> contourZ 8)
 <> layer3D (scatter3DPoints obsPts <> color3D (fromHex "#d62728") <> size3D 7)  -- Observed points
 <> camera (cameraIso 5) )
```

![RSM: Quadratic response surface + floor contour + observations](images/3d/rsm-surface-3d.svg)

> 3D SVG backend is mature. Interactive WebGL backend also available (→ 7-7).
> Surface z continuous color (colormap) not yet in WebGL (follow-up).

### 7-5. General-purpose 3D — Group colors, bubbles, bars, annotations, panes, log/aspect

Builder group extending surface-only 3D to general statistical plots (compose with `<>`):

| Builder | Type (what to pass) | Effect |
|---|---|---|
| `colorBy3D` | `ColRef -> Layer3D` | Color groups by categorical column + discrete legend (scatter/line) |
| `colorContinuousBy3D` | `ColRef -> Layer3D` | Color by numeric column (viridis) + colorbar (scatter) |
| `sizeBy3D` | `ColRef -> Layer3D` | Point size by numeric column (bubble) |
| `sizeRange3D` | `(Double, Double) -> Layer3D` | Bubble radius range |
| `bar3D` | `ColRef -> ColRef -> ColRef -> Layer3D` | 3D bar (x, y, z) |
| `barStyle3D` / `barWidth3D` | `BarStyle3D -> Layer3D` / `Double -> Layer3D` | Bar style (box/stick) / width |
| `errorBar3D` | `ColRef -> Layer3D` | Error bar column (SE) |
| `text3DPoints` | `[(Point3, Text)] -> Layer3D` | Label arbitrary 3D points (color=`color3D`, size=`size3D`) |
| `annotate3D` | `Point3 -> Text -> Layer3D` | Label single point |
| `pane3D` | `Bool -> VisualSpec3D` | Wall pane + grid on/off (default ON = mplot3d standard) |
| `xAspect3D` / `yAspect3D` / `zAspect3D` | `Double -> VisualSpec3D` | Box aspect ratio (mplot3d `set_box_aspect`) |
| `logScale3D` | `Bool -> Bool -> Bool -> VisualSpec3D` | Log scale x/y/z (ticks = powers of 10) |

```haskell
-- Group color + centroid labels (3D cluster visualization)
saveSVG3DBound "clusters.svg"
  ( df |>> ( layer3D (scatter3D "x" "y" "z" <> colorBy3D "cluster" <> size3D 5)
          <> layer3D (text3DPoints [(Point3 0 0 0.7, "A"), (Point3 2.2 1.5 1.7, "B")] <> size3D 16)
          <> camera (cameraIso 5) <> axisTitles3D "PC1" "PC2" "PC3" ) )
```

-- Continuous color + bubble + log-x (dose-response)
 <> ( scatter3D "dose" "time" "resp"

 <> logScale3D True False False    -- dose (x) to log

-- DoE bars + error bars + value labels

### 7-6. New marks — stem3D / quiver3D / trisurf

Three marks filling mplot3d's remaining types. All composable via `layer3D`, freely combined with camera / axisTitles3D / pane3D, etc. (7-5):

| Mark / builder | Type (what to pass) | Effect (comparison) |
|---|---|---|
| `stem3D` | `ColRef -> ColRef -> ColRef -> Layer3D` | Vertical line to base + endpoint (3D lollipop, mplot3d `stem`) |
| `stem3DPoints` / `stemBaseZ` | `[Point3] -> Layer3D` / `Double -> Layer3D` | Stems from point list / base z |
| `quiver3D` | `[(Point3, Vec3)] -> Layer3D` | Vector arrows at each point (mplot3d `quiver`) |
| `vecScale3D` | `Double -> Layer3D` | Arrow length scale (autoscale = 35% of cube) |
| `trisurf` | `[Point3] -> Layer3D` | Triangulate irregular points via Delaunay, form surface (mplot3d `plot_trisurf`) |

```haskell
-- stem3D (discrete series 3D lollipop)
saveSVG3DBound "stem.svg"
  ( df |>> ( layer3D (stem3D "t" "series" "value" <> color3D (fromHex "#d62728") <> stemBaseZ 0)
          <> camera (cameraIso 5) ) )

-- quiver3D (3D vector field, rotational flow)
saveSVG3D "quiver3d.svg"
  ( layer3D (quiver3D [ (p, rotFlow p) | p <- gridPts ] <> vecScale3D 1.0)
  <> camera (cameraIso 5) )

-- trisurf (irregular samples → surface)
saveSVG3D "trisurf.svg"
  ( layer3D (trisurf samplePts <> colormap3D) <> camera (cameraIso 5) )
```

| stem3D | quiver3D | trisurf |
|---|---|---|
| ![stem3D](images/3d/stem3d-a4.svg) | ![quiver3D](images/3d/quiver3d-a3.svg) | ![trisurf](images/3d/trisurf-a5.svg) |

### 7-7. Interactive 3D — WebGL backend (browser orbit)

3D from 7-4–7-6 work not just as static SVG/PDF/PNG but also via **browser-orbitable WebGL backend**. Spec unchanged, just swap save/display functions:

```haskell
import Hgg.Plot.ThreeD.Browser (showBrowser, saveHTML3D)
import Hgg.Plot.ThreeD.Bound   (showBrowser3DBound, saveHTML3DBound)

showBrowser   spec            -- Generate tmp HTML + launch browser (= interactive version of static saveSVG3D)
saveHTML3D    "out.html" spec -- Self-contained HTML for distribution (no external deps)
saveHTML3DBound "out.html" b  -- Df integration (pass BoundPlot3D from df |>> spec, resolves column refs)
```

WebGL backend has marks/features parity with SVG path:

| Category | WebGL support |
|---|---|
| Marks | scatter / line / wireframe / surface + **bar + error bars / text labels / stem / quiver / trisurf** |
| Colors | Group color (`colorBy3D`) + discrete legend / value map (`colorContinuousBy3D`) + colorbar / bubble (`sizeBy3D`) |
| Faces | surface/trisurf transparency (`alpha3D`), wall pane + grid |
| Axes | Log axes (`logScale3D`, labels = 10^value), box aspect (`xAspect3D` etc.), nice-tick numeric labels |

```haskell
-- Interactive display of group-color scatter from df (with legend)
showBrowser3DBound (df |>> ( layer3D (scatter3D "x" "y" "z" <> colorBy3D "cluster" <> size3D 8)
                          <> camera (cameraIso 5) ))
```

> Background is white like 2D/CPU plot (`tpBackground`). Axis numerics use nice ticks like 2D (`extendedBreaks`).
> **trisurf/surface z continuous color (colormap) not yet in WebGL** (scatter value maps are; surface colormap follow-up).
> WebGL lines fixed at 1px (quiver arrows / stem verticals / grid).

### 7-8. Other 3D builders {#more-3d}

Additional marks / builders not covered in 7-1–7-7 (all typed). Composable via `layer3D` / `VisualSpec3D` with `<>`:

| Function | Type (what to pass) | Purpose |
|---|---|---|
| `line3D` | `ColRef -> ColRef -> ColRef -> Layer3D` | 3D polyline (x, y, z) |
| `line3DPoints` / `bar3DPoints` | `[Point3] -> Layer3D` | Polyline / bars from points |
| `wireframe3D` | `[Point3] -> [(Int, Int)] -> Layer3D` | Vertices + edge indices for **edges-only** line mesh (no faces) |
| `shaded3D` | `Bool -> Layer3D` | **surface3D / trisurf faces** with Lambert shading (wireframe edges only, no effect) |
| `edgeOn` | `Layer3D` | Show edge lines on **surface/trisurf faces** (no args) |
| `edgeColor3D` / `edgeWidth` | `Color -> Layer3D` / `Double -> Layer3D` | Edge color / width |
| `width3D` | `Double -> Layer3D` | Line width |
| `colormapWith3D` | `[Text] -> Layer3D` | Explicit colormap as color list |
| `contourZ` / `contourX` / `contourY` | `Int -> Layer3D` | Projected contour count (floor=level set / walls=cross-sections, composable, walls auto-far from camera) |
| `axes3D` | `Axes3D -> VisualSpec3D` | Specify axes configuration |
| `autoAxes3D` | `[Layer3D] -> Axes3D` | Auto-construct axes from layers |
| `projection` | `Projection3D -> VisualSpec3D` | Projection method (`defaultPerspective`) |
| `width3DV` / `height3DV` | `Int -> VisualSpec3D` | Output px size |
| `cameraTop` / `cameraFront` / `cameraSide` | `Double -> Camera3D` | Viewpoint presets (family of `cameraIso`) |
| `defaultCameraZUp` / `defaultCameraYUp` | `Double -> Camera3D` | Default cameras (z-up / y-up) |
| `unBound3D` | `BoundPlot3D -> (Resolver, VisualSpec3D)` | Decompose df bundle |

> Up-vector is `zUp` / `yUp :: Vec3` (default z-up). Related types: `Vec3` / `Mat4` / `Camera3D` /
> `Projection3D` / `BarStyle3D` / `Axes3D` / `Point3` / `Layer3D` / `VisualSpec3D`.

```haskell
-- Wireframe = vertices + edge indices for edges-only line mesh (e.g. cube = 8 vertices, 12 edges).
-- Has no faces, so shaded3D/colormap3D have no effect (filled surfaces need surface3D / trisurf).
saveSVG3D "wire.svg"
  ( layer3D (wireframe3D verts edges <> color3D (fromHex "#2ca02c") <> width3D 1.5)
 <> camera (cameraFront 5) )      -- cameraIso/cameraTop/cameraSide also available
```

![3D wireframe (cube, edges-only line mesh)](images/3d/wireframe3d-cube.svg)

> ⚠️ `wireframe3D` draws **edges only** (= frame lines). For gridline "wireframe surface" look,
> the result is still a line mesh without fill. To get filled surfaces (continuous color, shading), use `surface3D` (7-1) / `trisurf` (7-6).
